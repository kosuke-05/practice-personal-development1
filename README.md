## システム概要
- ログインユーザー（社員と仮定）ごとに登録したタスクを管理できるタスク管理アプリ
- 認証に成功したユーザーが自分の登録したタスクを閲覧・編集・削除することが出来る

## 主な機能
- タスクのCRUD機能
- タスク検索（社員名・部署名のいずれか部分合致で検索が可能）
- ユーザー認証（ログイン / ログアウト）
- ログインユーザーが登録したタスクのみを表示する詳細画面
- ページネーション機能（1ページにつき5タスクまで表示できる仕様）

## 使用技術
- フロントエンド：React / Next.js / TypeScript
- 状態管理：Zustand / Context / useState
- データ取得：React Query
- モックAPI：MSW
- バージョン管理：Git / GitHub

## 各ページ・ダイアログ情報
### ホーム画面
- 全タスク（テーブルで表示）
- AppBar（検索機能とログイン・ログアウトを行うことが可能）
- Drawer（ホーム画面・タスクの新規登録・詳細画面・設定メニュー（仮の見た目）を実装）

### 新規登録画面
- タスク名・タスク詳細・タスク優先度・タスク進捗度・期限日・社員名・部署名を入力

### 詳細画面
- ログイン済みのユーザーが登録したタスクのみが表示される
- 詳細画面でのみタスクの編集・削除を行うことが可能となる

### ユーザー新規登録ダイアログ
- 社員名・部署名・メールアドレス・パスワードを入力して登録
- ユーザー新規登録が完了後、ログインしてログイン済みの状態となる

### ユーザーログインダイアログ
- メールアドレス・パスワードを入力してログイン
- ログイン成功後、ログアウトするまでユーザー新規登録は出来ない状態にしている

### ログインを促すダイアログ
- 未ログイン状態で詳細メニューを押下した場合に表示
- ログインが成功していたら、ダイアログは表示されずに詳細画面に遷移が可能

## 工夫した点
### コンポーネントを極力使いまわししている
- タスク登録画面とタスク編集画面
- ボタンも類似したロジックを持つ場合は使いまわしを意識している

### ロジックとUIの責務を分離
- 責務分離を重要視して実装しており、コンポーネントファイルには基本的にコンポーネント定義しか行っていない
- ボタン押下後に呼び出される関数などは、大半ロジックコンポーネント側に定義している（場合によって、一部コンポーネントファイル内に定義している）

### レイヤーごとにファイルを分離
- API通信の一連の処理をUI層にまとめて定義するのではなく、hooks層・api層・mswと分離させて役割によってファイルを分けている
- page.tsxにまとめてコーディングせず、ロジックコンポーネントとアプリケーションコンポーネントでファイル分割を行い、各々の役割を満たした処理のみ定義するよう心掛けている
